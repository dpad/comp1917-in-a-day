Chapter 6: Functions

Functions

Suppose in your program you had a need to calculate the value of a number to the power of another number. This is easy enough to code, however what if there were two or more parts in your code when this is required? Of course, you could copy all the code for finding powers to this new section, but it is messy, and a far more elegand approach is to write a function.

You have already used several functions. scanf, printf, sizeof and main are all functions that you will have seen before. Functions can be thought of in a similar way to functions in mathematics. That is, they take several values as input (these are called arguments), do something with them, and possibly return a value. The number of arguments can be zero, and there can be as many as you like. Functions can return at most one value, however sometimes it is unnecessary to return anything (e.g. functions for printing).

Here is an example of code with a power function:
  1. #include <stdio.h>
  2.  
  3. int power(int base, int index);
  4.  
  5. int main(int argc, char *argv[]) {
  6.  
  7.     int baseInput, indexInput;
  8.     int answer;
  9.  
  10.     printf("Enter base: ");
  11.     scanf("%d", &baseInput);
  12.  
  13.     printf("Enter index: ");
  14.     scanf("%d", &indexInput);
  15.  
  16.     answer = power(baseInput, indexInput);
  17.     printf("%d^%d = %d\n", baseInput, indexInput, answer);
  18.  
  19.  
  20.     return 0;
  21. }
  22.  
  23. int power(int base, int index) {
  24.     int answer = 1;
  25.     int i;
  26.  
  27.     for (i = 0;i<index;i++) {
  28.         answer *= base;
  29.     }
  30.  
  31.     return answer;
  32. }

Some things to note


On line 3, there is a prototype for the power function. This is essentially the first line of the function definition but without the open parenthesis and ending in a semicolon.
When defining a function, begin with the return type (the type of the value that will be returned). In this case it is int. If a function does not return anything, begin the line with "void".
Then comes the name of the function. This is the name that you use when calling the function. The name of this function is "power".
Finally, there is a list of arguments. The names of arguments given here must be used to refer to them inside the function, however they have no influence on the names of variables or values passed to the function when it is called.
If the function has a return type, it must end with a return line, returning a value of that type.

Some definitions:
Call in this context refers to when the function is activated. This is done by using the name of the function. It can be used in the same way as a variable, value or expression (a function call is an expression) to access the return value.
Pass refers to giving values as input to a function. This is done by placing a comma seperated list of values in brackets after the function name. If the function takes no values, use an empty set of brackets.

One could accurately say that in the above code, baseInput and indexInput are passed to power. Power is called on line 16 and its return value is stored in answer.

Passing by Value

Let's do a quick experiment. Consider the code below.
  1. #include <stdio.h>
  2.  
  3. void swap(int a, int b);
  4.  
  5. int main(int argc, char *argv[]) {
  6.  
  7.     int a = 1;
  8.     int b = 2;
  9.  
  10.     printf("a: %d\nb: %d\n", a, b);
  11.  
  12.     printf("Attempting swap...\n");
  13.     swap(a, b);
  14.  
  15.     printf("a: %d\nb: %d\n", a, b);
  16.  
  17.     return 0;
  18. }
  19.  
  20. void swap(int a, int b) {
  21.     int tmp = a;
  22.     a = b;
  23.     b = tmp;
  24. }

Clearly, the swap function swaps the values of the two arguments right?
Here is the output:
a: 1
b: 2
Attempting swap...
a: 1
b: 2

So what happened?


C uses something called Pass by Value when passing arguments to functions. This means that a local "copy" of the variables is used inside the functions. The bottom line is, any changes made to the argumensts will not affect the original values (it will only affect the copies inside the function). These copies effectively disappear after the function returns, and the only value that can be accessed is its return value.
