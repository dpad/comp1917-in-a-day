Chapter 7: Arrays

Ideal Arrays

Let's begin by looking at using arrays for the purpose for which they were intended. Ideal is in the same sence as the ideal capacitor. For now we will not look at how arrays actually work - what happens behind the scenes. This will be covered soon.

An array is a list of variables of a single type. (ie. they are all integers, or all chars, etc) They are declared in the same way to variables, but with square brackets containing the size of the array. (size of array: how many values can be stored) When accessing elements of an array, use the name of the array followed by square brackets containing the index of the desired element.

Here is an example of code using an array.
  1. #include <stdio.h>
  2.  
  3. #define SIZE 4
  4.  
  5. int main(int argc, char *argv[]) {
  6.  
  7.     int list[SIZE];
  8.     int i;
  9.  
  10.     list[0] = 5;
  11.     list[1] = 2;
  12.     list[2] = 8;
  13.     list[3] = 3;
  14.  
  15.     for (i = 0;i<SIZE;i++) {
  16.         printf("list[%d] = %d\n", i, list[i]);
  17.     }
  18.  
  19.     return 0;
  20. }

output
list[0] = 5
list[1] = 2
list[2] = 8
list[3] = 3

Note that the highest index you can give to an array is one LESS than its size.

Multi-dimensional Arrays

You can have arrays of as many dimensions as you like. For example a 2D array is an array of arrays. You can think of it as a 2D grid of values. When declaring a 2D array, use a pair of square brackets with the "dimensions" of the array. Adding more dimensions is the same process, however you seldom use more than 2 dimensions.

The following example uses a 2D array to print a circle.
  1. #include <stdio.h>
  2.  
  3. #define SIZE 30
  4. #define RAD 10
  5.  
  6. int power(int base, int index);
  7.  
  8. int main(int argc, char *argv[]) {
  9.  
  10.     int grid[SIZE][SIZE]; //the grid that will store the circle
  11.  
  12.     int i, j; //counters
  13.  
  14.     int x, y; //for storing x and y values of points
  15.  
  16.  
  17.     //store the circle
  18.     for (i = 0;i<SIZE;i++) {
  19.  
  20.         y = i - SIZE/2;
  21.  
  22.         for (j = 0;j<SIZE;j++) {
  23.  
  24.             x = j - SIZE/2;
  25.  
  26.             //check if a the point lies in the circle
  27.             if (power(x, 2) + power(y, 2) <= power(RAD, 2)) {
  28.                 //if so store a 1
  29.                 grid[j][i] = 1;
  30.             } else {
  31.                 //otherwise store a 0
  32.                 grid[j][i] = 0;
  33.             }
  34.         }
  35.     }
  36.  
  37.     //print a circle of hashes
  38.     for (i = 0;i<SIZE;i++) {
  39.         for (j = 0;j<SIZE;j++) {
  40.             if (grid[j][i] == 1) {
  41.                 printf("#");
  42.             } else {
  43.                 printf(" ");
  44.             }
  45.         }
  46.         printf("\n");
  47.     }
  48.  
  49.  
  50.  
  51.     return 0;
  52. }
  53.  
  54. int power(int base, int index) {
  55.     int answer = 1;
  56.     int i;
  57.  
  58.     for (i = 0;i<index;i++) {
  59.         answer *= base;
  60.     }
  61.  
  62.     return answer;
  63. }

This essentially builds an array of flags for each point, then uses the array to draw a circle. There are loads of uses for 2D arrays. Any time when you need to store data about a grid, you should probably use a 2D array.

The above code also shows how a for loop nested inside another for loop can be used to traverse a 2D array.

More array notes

You will hear the term "bounds" used in the context of arrays. The bounds of an array is the range of indices that can be used with the array. If you use an index that is too high (or negative), it is said to be outside the bounds of the array. If you do this in your code, it will still compile without warnings or errors. C doesn't care, as as far as it's concerned this is perfectly valid code. However when the program is running, attempting to change array elements outside the bounds may result in a segmentation fault. You should therefore be very careful with how you access data in arrays.

Also, when you declare an array, the values do not get changed at all. They remain as they were before the memory was assingned to the array. Yhis is a consequence of c being such a low level language. At no point should you access a value in an array that you didn't first assign.
